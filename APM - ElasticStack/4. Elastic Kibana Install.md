## 2) Chuẩn bị máy chủ (áp dụng DEV/TEST & Prod)

**Thông số tham chiếu cho DEV/TEST 3 VM**

* vCPU: 4–8
* RAM: 16–32 GB (ES heap ~ 1/2 RAM nhưng **≤ 32GB**)
* Disk: SSD NVMe, tách: `/var/lib/elasticsearch` riêng; thêm disk cho repo nếu dùng FS.
* OS: Rocky 9.

**System tuning tối thiểu**

* `vm.max_map_count=262144`
* Tắt swap, hoặc bật `bootstrap.memory_lock: true`
* `nofile` ≥ 65536; `nproc` ≥ 4096
* Đồng bộ **NTP/chrony**
* Mở port: 9200 (HTTP), 9300 (transport), 5601 (Kibana), 8220 (Fleet), 5044 (Logstash beats) nếu dùng.

> Gợi ý chuẩn hoá qua Ansible role `elastic_preflight`.

```bash
# Linux sysctl & limits
cat << 'EOF' | sudo tee /etc/sysctl.d/99-elastic.conf
vm.max_map_count=262144
EOF
sudo sysctl --system

# Disable swap (khuyến nghị)
sudo swapoff -a
sudo sed -i.bak '/\sswap\s/s/^/#/' /etc/fstab

# Limits
cat << 'EOF' | sudo tee /etc/security/limits.d/99-elastic.conf
* soft nofile 65536
* hard nofile 65536
* soft nproc  4096
* hard nproc  4096
EOF
```

## 5) Bật security, TLS, và cluster 3 nút

> Elastic 9.x bật **security** mặc định. Ta tạo **cert nội bộ** và **enrollment token** để gia nhập node & Kibana.

### 5.1 Tạo chứng chỉ nội bộ trên **es-dev-3** (dùng cho cả cluster)

```bash
sudo /usr/share/elasticsearch/bin/elasticsearch-certutil ca --pem -out /etc/elasticsearch/certs/elastic-stack-ca.zip
sudo /usr/share/elasticsearch/bin/elasticsearch-certutil cert --name es-dev-wildcard \
  --dns es-dev-1,es-dev-2,es-dev-3 \
  --ip 10.10.10.11,10.10.10.12,10.10.10.13 \
  --ca-cert /etc/elasticsearch/certs/elastic-stack-ca.zip \
  --pem -out /etc/elasticsearch/certs/es-dev-certs.zip
sudo mkdir -p /etc/elasticsearch/certs && sudo unzip -o /etc/elasticsearch/certs/*.zip -d /etc/elasticsearch/certs
sudo chown -R elasticsearch:elasticsearch /etc/elasticsearch/certs
```

### 5.2 Cấu hình `elasticsearch.yml` trên **mỗi node**

```yaml
# /etc/elasticsearch/elasticsearch.yml
cluster.name: es-devtest-9
node.name: es-dev-1   # thay theo node
node.roles: [ master, data, ingest ]
path.data: /var/lib/elasticsearch
path.logs: /var/log/elasticsearch
network.host: 0.0.0.0
http.port: 9200
transport.port: 9300

# Peers
discovery.seed_hosts: ["10.10.10.11","10.10.10.12","10.10.10.13"]
cluster.initial_master_nodes: ["es-dev-1","es-dev-2","es-dev-3"]

# Security + TLS
xpack.security.enabled: true
xpack.security.http.ssl.enabled: true
xpack.security.http.ssl.key: "/etc/elasticsearch/certs/es-dev-wildcard.key"
xpack.security.http.ssl.certificate: "/etc/elasticsearch/certs/es-dev-wildcard.crt"
xpack.security.http.ssl.certificate_authorities: ["/etc/elasticsearch/certs/ca.crt"]

xpack.security.transport.ssl.enabled: true
xpack.security.transport.ssl.key: "/etc/elasticsearch/certs/es-dev-wildcard.key"
xpack.security.transport.ssl.certificate: "/etc/elasticsearch/certs/es-dev-wildcard.crt"
xpack.security.transport.ssl.certificate_authorities: ["/etc/elasticsearch/certs/ca.crt"]

# Khuyến nghị
bootstrap.memory_lock: true
```

> Sao chép thư mục `/etc/elasticsearch/certs` từ es-dev-3 sang es-dev-1/2 (scp/ansible), đồng bộ quyền.

### 5.3 Start dịch vụ & đặt mật khẩu built-in

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now elasticsearch

# Thiết lập mật khẩu cho users built-in (elastic, kibana_system, logstash_system, ...)
sudo /usr/share/elasticsearch/bin/elasticsearch-setup-passwords auto
# Ghi lại output để dùng cho Kibana & Fleet
```

## 6) Cài Kibana 9.x (trên es-dev-3)

```bash
# APT
sudo apt install -y kibana || true
# YUM
sudo dnf install -y kibana || true
```

Cấu hình `/etc/kibana/kibana.yml`:

```yaml
server.host: 0.0.0.0
server.publicBaseUrl: "https://kibana.dev.local"   # nếu có reverse proxy/LB
elasticsearch.hosts: ["https://10.10.10.11:9200","https://10.10.10.12:9200","https://10.10.10.13:9200"]
elasticsearch.username: "kibana_system"
elasticsearch.password: "<PASSWORD_TU_STEP_5.3>"

elasticsearch.ssl.certificateAuthorities: ["/etc/elasticsearch/certs/ca.crt"]
```

Khởi động Kibana:

```bash
sudo systemctl enable --now kibana
```

Đăng nhập Kibana bằng user `elastic` ⇒ vào **Stack Management → Users/Roles → tạo tài khoản quản trị riêng**.
